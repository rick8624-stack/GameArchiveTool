# 项目需求：GameArchiveTool - 游戏压缩包批量处理工具

## 项目背景

我有大量游戏压缩包（Xbox360游戏等），存放在多层文件夹结构中，需要一个Windows GUI工具完成批量预处理、解压、重命名的完整流水线。

## 技术栈要求

- Python 3.10+，GUI用tkinter（标准库，无额外依赖）
- 解压核心调用7z.exe子进程（路径可配置，默认 C:\Program Files\7-Zip\7z.exe）
- 打包目标：后续用pyinstaller打成单exe，代码结构需考虑此点（资源路径用sys._MEIPASS兼容处理）
- 单一主文件或清晰的多模块结构均可，优先可读性和可维护性
- 所有文件I/O和字符串处理必须正确处理中文（UTF-8），文件名操作用pathlib

## 功能需求

### 模块一：预处理（文件名清理）

- 递归扫描指定根目录，找出文件名末尾带多余字符的文件并批量重命名
- 默认规则：去掉文件名末尾的"删"字，如 `游戏.7z.001删` → `游戏.7z.001`
- 规则可配置（用户可在UI中修改要去除的后缀字符/正则）
- 必须有预览模式：先展示"原名 → 新名"列表，用户确认后才执行

### 模块二：批量解压（核心功能）

- 递归扫描根目录下所有文件夹中的压缩文件
- 支持格式：zip、rar、7z，以及分卷压缩（.7z.001、.zip.001、.part1.rar、.r00旧式分卷）
- 分卷只处理首个分卷文件（.001/.part1），跳过后续分卷，由7z自动串联
- 解压目标位置：默认解压到压缩包所在文件夹内（可选：解压到以压缩包名命名的子文件夹）
- 密码池机制：
  - 先尝试无密码解压
  - 失败后按密码池顺序逐个尝试（用 7z t 测试模式验证密码，验证通过才真正解压）
  - 密码池带命中计数，命中次数多的密码自动前移（持久化到配置文件）
  - 密码池在UI中可增删改，保存到config.json
- 解压成功后可选删除原压缩包（含分卷全部文件），默认关闭，UI中有开关
- 失败的压缩包记入失败清单，不中断整体流程

### 模块三：批量重命名（编号对照表）

- 读取CSV对照表（两列：编号,英文名），将文件夹名中的中文名替换为英文名
- 文件夹命名格式：`001.中文名` → `001.英文名`，编号部分保留原样（含前导零）
- 编号匹配时去除前导零（文件夹"001"匹配对照表的"1"）
- 支持字母编号（XB001、K001、KX01等格式）
- 新文件夹名自动清除Windows非法字符 \/:*?"<>|
- 必须有预览模式，确认后执行
- 未匹配到对照表的文件夹列入待处理清单展示给用户

### 模块四：GUI与进度可视化

- 主窗口布局：
  - 顶部：根目录选择（输入框+浏览按钮，支持拖拽文件夹进窗口）
  - 选项卡或分区：预处理 / 解压 / 重命名 三个功能区
  - 中部：双级进度条（总进度：已处理数/总数；当前文件进度：解析7z的-bsp1输出流获取实时百分比）
  - 底部：实时日志窗口（滚动文本框，成功绿色、失败红色、信息灰色）
  - 状态栏：成功/失败/跳过计数
- 所有耗时操作必须在工作线程执行，UI线程不能阻塞（用threading + queue向UI推送进度事件）
- 提供"停止"按钮，可中断当前批量任务（当前文件处理完后停止）

### 模块五：状态持久化与断点续传

- config.json：保存7z路径、密码池（含命中计数）、上次使用的根目录、各开关状态
- progress.json：批量任务执行中，每完成一个压缩包记录其路径；重新启动任务时询问是否跳过已完成项
- 任务结束后可导出处理报告CSV（文件路径、操作类型、结果、使用的密码、耗时）

## 边界情况处理

- 压缩包内文件与目标目录已有文件重名：7z用-y参数覆盖，但在日志中标注
- 分卷不完整（缺.002等）：7z测试会失败，日志标注"疑似分卷不完整"而非单纯"密码错误"（可通过先列出同名分卷数量辅助判断）
- 路径过长（Windows 260字符限制）：日志警告并跳过
- 压缩包本身损坏：归入失败清单，标注错误类型
- 空文件夹、无压缩包的文件夹：静默跳过
